## 1. Features

### 1.1 Folder Upload Hardening

OS 폴더 drag·drop과 `Upload folder` picker 양쪽 경로가 동일하게 강화되었습니다.

- 재귀 깊이 32단계 cap. 초과 시 1회 경고 toast + 부분 업로드 보존.
- Dot-prefix 세그먼트(`.git`, `.DS_Store`, 중간 경로의 `.hidden/` 등)는 모든 경로 깊이에서 정적 스킵.
- `FileSystemDirectoryReader.readEntries` try/catch — 일부 디렉토리 읽기 실패 시 수집된 파일은 보존하고 계속 진행.
- 드롭 종료 시 `Queued N file(s) for upload` 또는 `No uploadable files in dropped folder` 요약 알림.
- `window.__nasTestEnqueueFolder` DEV-only harness 노출 — `import.meta.env.DEV` 가드로 prod 빌드에서 tree-shake.
- Backend regression test 추가 (`TestTusUploadCreatesNestedFolders`): 3·5단계 중첩과 root-level 업로드에서 `os.MkdirAll(0o755)` 동작 검증.

### 1.2 File Details (Size, Modified, Inspector)

- Backend `fileEntry` JSON에 `size: int64`와 `modifiedAt: RFC3339` 필드 추가. `entry.Info()` 단일 호출로 추가 stat 부담 없음.
- List view: `Size`(우측 정렬, 2-decimal precision, 폴더는 `—`)와 `Modified`(`just now`/`Nm`/`Nh`/`Nd`/`Mon D, YYYY`) 컬럼. Modified cell의 `title` 속성에 full timestamp.
- Grid view: 파일 tile의 `title`이 `"이름\n1.00 MB · 2h"`.
- Inspector 사이드 패널(`w-[360px]`, UploadPanel 패턴): icon+name / Type / Size / Modified / Path. 트리거 방식은 §3.3 참조.

### 1.3 Activity Graph & Git-style Timeline

- `aggregate.ts` 모듈: `dailyCounts(entries, days=30)` (local-date bucket, zero-fill)와 `typeDistribution(entries)` (OPEN→VIEW fold, ratio guard).
- `ActivityGraph.svelte` 배너: Last-30-days sparkline (accent yellow `#fabd2f`, height 48, ceiling = local max) + Distribution 누적 바 + 범례 pill(`title="N · X%"`).
- 타임라인은 day-section sticky header로 묶임(`Today / 2026-05-14 / 10 events`). 각 row는 3-컬럼: `HH:MM` mono · activity dot + author initial chip · activity type + `[TARGET]` chip + author name + path.
- `MOVE` 항목은 `parse()` helper가 분해해 `/from` → `/to` 화살표(lucide `ArrowRight`) 렌더링. `UPLOAD`/`CREATE`/`DELETE`는 단일 path mono.
- Side-fix: `ActivityLog.load()`가 `{data: [...]}` shape의 백엔드 응답을 무시하던 silent bug를 함께 수정.

### 1.4 Music and Video Library Tabs

- Backend `GET /mediaLibrary?kind=audio|video` 추가. `filepath.WalkDir`로 NAS 재귀, dot-prefix 디렉토리는 `fs.SkipDir`. `MEDIA_LIB_LIMIT` (default 5000) 도달 시 walk early-exit + `X-Library-Truncated: true` 응답 헤더.
- `mediaEntry` 페이로드는 `loc`(디렉토리)와 `name`(파일명)을 분리해 기존 `FilePayload` 형식 그대로 viewer 탭에 전달 가능.
- `MusicLibrary.svelte`: loc별 compact 그룹 리스트 + 검색 필터 + IntersectionObserver 페이지네이션 (200 + 200).
- `VideoLibrary.svelte`: 16:9 그리드 (auto-fill 220px), `film` 아이콘 placeholder, IntersectionObserver 페이지네이션 (60 + 60).
- 클릭 시 기존 `MediaViewer` 탭이 열림. 사이드바·TabBar·QuickOpen의 `KIND_TO_ICON` 매핑도 함께 갱신.

### 1.5 Mobile UI Support

전반에 걸친 반응형 변환. 기존 `ui.svelte.ts`의 `isMobile`/`breakpoint`/`viewportWidth/Height`를 재사용하고 `drawerOpen` 상태를 확장하는 방식으로 통합.

- `.h-app` 유틸리티(`100vh` fallback + `100dvh` override) — iOS Safari URL바 클리핑 회피.
- `(app)/+layout.svelte` 그리드: `grid-cols-[1fr] md:grid-cols-[auto_1fr]`. 사이드바는 `hidden md:contents`로 모바일에서 DOM 흐름에서 제외.
- `MobileDrawer.svelte` (신규): `fixed inset-0 z-50` 백드롭 + 좌측 w-64 패널. `documentElement.style.overflow = 'hidden'`으로 body scroll lock. Esc 키 + md 이상 리사이즈 시 자동 close.
- `Header.svelte` 모바일 condense: 햄버거 추가(`menu` icon, `md:hidden`), 로고·사용자명·Quick-open 라벨은 `hidden md:flex`.
- `Toolbar.svelte` + `ToolbarOverflow.svelte`: Up / Upload(아이콘) / View toggle만 노출, Upload folder / New folder / Search inline은 `...` 오버플로우 드롭다운으로 이동. Search는 별도 토글 버튼으로 풀폭 바 노출.
- `TabBar.svelte`: 모바일에서 width shrink (`min-w-[100px] max-w-[140px]`), close X 상시 표시(`md:opacity-0 md:group-hover:opacity-100`), 드래그 재정렬 비활성(`draggable={!ui.isMobile}`).
- `UploadPanel.svelte` / `Inspector.svelte`는 모바일에서 bottom sheet 변형(`fixed inset-x-0 bottom-0 max-h-[70dvh] rounded-t-lg pb-[env(safe-area-inset-bottom)]`).
- `QuickOpen.svelte`는 모바일에서 fullscreen. `visualViewport.height` 추적으로 소프트 키보드가 올라와도 검색 입력란이 가시 영역에 유지.
- 터치 입력: `long-press.ts` Svelte action(500ms 홀드, 8px 이동 시 취소)을 도입해 컨텍스트 메뉴 호출 경로 일원화. 엔트리는 `draggable={!ui.isMobile}`로 모바일 HTML5 드래그 비활성.
- Viewer 반응형: `MonacoViewer`의 `wordWrap`이 모바일에서 `on`. `VideoControls`·`PdfViewer`의 보조 컨트롤은 `...` 오버플로우로 접힘.
- 동일 outside-click 패턴이 세 컴포넌트에서 중복된 것을 코드 리뷰 중 발견하여 `frontend/src/lib/actions/click-outside.ts` Svelte action으로 추출.

### 1.6 Marquee and Multi-Select

탭별 격리된 다중 선택과 마퀴 사각형 선택, 배치 작업을 추가.

- 선택 상태는 `ExplorerPayload.selection?: string[]`에 저장(마지막 원소가 Shift 범위 anchor).
- 클릭 modifier 우선순위: `Shift > Ctrl/Cmd > none`. `setSingle` / `toggleEntry` / `selectRange` / `clear`는 `frontend/src/lib/components/Explorer/selection.ts` 순수 함수.
- 키보드: `Ctrl/Cmd+A` (전체), `Esc` (해제), `Delete` (배치 — `Promise.allSettled` 후 `Deleted N (M failed)` 요약 토스트). 편집 가능한 input 위에서는 모두 단락 처리.
- 마퀴: pointer-down·move·up + 3-layer guard(`data-marquee-canvas` ancestor check / 스크롤바 offset reject / editable target reject) + 32px 엣지 근접 시 8px/frame 자동 스크롤 + entry rect re-snapshot. 4px dead zone으로 일반 클릭과 구분.
- 드래그 페이로드는 `{items: Array<{name,isFolder}>, sourceLoc}` 배치 형식. legacy single-entry shape는 `readPayload`가 자동 wrap (back-compat).
- 모바일 선택 모드: long-press 진입 → tap = toggle → 상단 바 `N selected` + Cancel(X) + Delete(휴지통).

### 2.2 Explorer Reactivity Loop

증상은 페이지 로드 직후 "Loading…" 표시가 사라지지 않고 콘솔에 `effect_update_depth_exceeded`. 삭제·이름변경 후에도 entries가 갱신되지 않음.

원인: 다중 선택 기능 도입 시 추가된 `$effect`가 `updateSelection`을 통해 `tabs.list`에 write → `TabContent`가 `<Explorer loc={tab.payload.loc}>` prop 표현식을 재평가 → Svelte 5가 prop 변경으로 인지 → effect re-fire → 무한 사이클.

수정:

- Inspector `selected` 상태를 `$state + $effect mirror`에서 순수 `$derived`로 변경(write 경로 제거).
- loc-change `$effect`에서 `updateSelection([])` 제거. `navigateTo` 등 네비게이션 핸들러가 `payload.selection = []`을 명시적으로 작성.
- title-update tabs.write는 `untrack()`으로 감쌈.

이 수정 이후 folder rename·delete의 자동 refresh가 의도대로 동작.

## 3. User-Interface Refinements

### 3.1 Typography — IBM Plex Sans, Weight 550 Floor

- Latin variable font(`@fontsource-variable/ibm-plex-sans`, axis 100–700)와 Korean static font(`@fontsource/ibm-plex-sans-kr`, weights 500/600/700)를 함께 추가.
- `--font-sans: "IBM Plex Sans Variable", "IBM Plex Sans KR", "Inter", ui-sans-serif, ...`.
- `html` 기본 weight를 `550`으로 설정(`font-variation-settings: 'wght' 550`). 한글은 가장 가까운 정적 weight인 600으로 snap.
- Tailwind 헬퍼 weight 재정의: `font-medium → 600`, `font-semibold → 650`, `font-bold → 700`. 표준 400~500 weight는 더 이상 사용되지 않으며 550이 새 floor.
- `font-mono`, `code`, `pre`, `kbd`, `samp`는 FiraD2 유지 + `font-variation-settings: normal`로 정적 폰트 그대로.

### 3.2 Background Context Menu

File canvas의 빈 공간에서 우클릭하면 폴더 레벨 메뉴가 등장합니다.

- 항목: `New folder` · `Upload files` · `Upload folder` · `Refresh`.
- 마퀴와 동일한 3-layer guard 재사용 — entry 위·편집 가능한 input 위에서는 발화하지 않음.
- 엔트리 컨텍스트 메뉴와 상호 배타(`bgMenuOpen` ↔ `menuOpen`).

### 3.3 Inspector — Explicit Trigger

이전에는 단일클릭으로 Inspector가 열려 화면을 가렸습니다. 이제는 명시적 선택에만 반응합니다.

- 단일클릭: 선택 ring(`bg-bg-elevated ring-1 ring-accent`)만 표시. Inspector는 열리지 않음.
- 우클릭 → `Details` (lucide `Info` 아이콘) → Inspector 오픈.
- Inspector가 열린 동안 다른 entry를 클릭해도 swap되지 않음 — 명시적 close(X) 또는 폴더 이동만 닫음.
- 삭제·이름변경·필터로 inspected entry가 사라지면 `$derived inspectedExists` + `$effect`(`untrack`)으로 자동 close. write-back 사이클 차단.

- Dead code 제거: `viewerIconName()` (`registry.ts`), `FilesStore`의 unused methods/state.
- `FileGrid` / `FileList`의 drag-drop 핸들러 중복을 `dragEntry` Svelte 5 action으로 통합.

## 6. Known Limitations and Follow-ups

다음 항목은 본 릴리스 범위 밖이며 후속 작업으로 분리되어 있습니다.

- **Notifications Toaster 미구현**: `notifications.svelte.ts`는 큐를 채우지만 이를 렌더링하는 컴포넌트가 없어 모든 toast가 silent. 별도 PR로 처리 예정.
- **iOS Safari 실기 검증**: `safe-area-inset-bottom`과 `visualViewport` 키보드 추적은 데스크탑 브라우저 자동화 환경에서 완전 에뮬레이션 불가. 실기기 검증 보류.
- **Explorer.svelte 분해**: 단일 컴포넌트가 637 lines로 증가. 마퀴·단축키·선택·드래그 이동을 네 개의 composable로 분리하는 작업을 백로그 항목으로 분리.
