# Markdown Viewer (View/Edit Toggle) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `.md` files currently open in Monaco editor as syntax-highlighted source. Add a **rendered HTML view** (Tailwind typography styled, sanitized) with a **view ↔ edit toggle** in the toolbar so users can read formatted Markdown by default and switch into the existing Monaco editor to modify.

**Architecture:** Introduce `MarkdownViewer.svelte` as a new wrapper component. It owns a `mode: "view" | "edit"` state (defaults to `"view"`), fetches the file content once, and either renders sanitized HTML via `marked` + `DOMPurify` (view mode) or delegates to the existing `MonacoViewer.svelte` (edit mode). `TabContent.svelte` dispatches `.md` files to `MarkdownViewer` instead of `MonacoViewer` directly. Other text formats (`.txt`, `.json`, etc.) continue routing straight to Monaco.

**Tech Stack:** Svelte 5 runes, Tailwind 4 with `@tailwindcss/typography` plugin (Markdown body styling), `marked` v12+ (CommonMark parser), `DOMPurify` v3+ (XSS sanitizer), existing `MonacoViewer.svelte` for edit mode.

**Visual Design:**
- View mode: `bg-bg-base` background, scrollable, max content width `max-w-[72ch]` centered, padding `p-8`, Tailwind `prose prose-invert` classes with project-tuned overrides (link color = `text-fg-link`, code background = `bg-bg-elevated`, blockquote border = `border-l-border-strong`)
- Code blocks: use Tailwind defaults; full Monaco-style highlighting deferred to v2 (would need `highlight.js` or `shiki`)
- Toolbar: 36px tall top bar matching MonacoViewer's existing toolbar, two-segment toggle "View | Edit" right-aligned, active segment uses `bg-bg-selected text-fg-accent`

**Testing Approach:** No frontend test framework — spec-driven manual verification per task. Type gate: `npm run check`. Per-implementation gate: `/code-review`.

**Out of scope (separate / future):**
- Syntax highlighting inside fenced code blocks (just `<code>` styling, no per-language highlight) — v2
- Math (KaTeX), Mermaid, footnotes — v2
- Live preview (split view) — UX decision; user said "view/edit toggle" specifically
- Server-side rendering for SEO — N/A (auth-gated app)
- Markdown linting / validation — out of scope

- [ ] **Step 1: Write the component**

```svelte
<!-- frontend/src/lib/components/Viewers/MarkdownRender.svelte -->
<script lang="ts">
  import { marked } from "marked";
  import DOMPurify from "dompurify";

  interface Props {
    content: string;
  }

  let { content }: Props = $props();

  const html = $derived(DOMPurify.sanitize(marked.parse(content, { async: false }) as string));
</script>

<article
  class="prose prose-invert max-w-[72ch] mx-auto p-8 text-fg-primary
         prose-headings:text-fg-primary
         prose-a:text-fg-link prose-a:no-underline hover:prose-a:underline
         prose-strong:text-fg-primary
         prose-code:text-fg-accent prose-code:bg-bg-elevated prose-code:px-1 prose-code:py-0.5 prose-code:rounded prose-code:before:content-[''] prose-code:after:content-['']
         prose-pre:bg-bg-elevated prose-pre:border prose-pre:border-border-default
         prose-blockquote:border-l-border-strong prose-blockquote:text-fg-secondary
         prose-hr:border-border-default
         prose-th:text-fg-primary prose-th:border-border-default

</article>
```

## Task 4: `MarkdownViewer.svelte` — toggle wrapper

**Files:**
- Create: `frontend/src/lib/components/Viewers/MarkdownViewer.svelte`

**Spec:**

| Behavior | Expected |
|---|---|
| Default mode | `"view"` (rendered HTML) |
| Toggle button | Two segments "View | Edit", right-aligned in toolbar |
| Click "Edit" | Mode = `"edit"`, shows MonacoViewer with same loc/name/tabId |
| Click "View" | Mode = `"view"`, re-fetches content (or uses cached), shows MarkdownRender |
| Top toolbar | 36px, matches MonacoViewer's filename + lang badge style, plus the segmented toggle |
| Content fetch | Same endpoint as MonacoViewer — `/server/getTextFile` with JSON envelope |
| Loading state | "Loading…" text centered while content arrives |
| Error state | Reuse notifications store with `error()` call (matches MonacoViewer pattern) |

- [ ] **Step 1: Write the component**

```svelte
<!-- frontend/src/lib/components/Viewers/MarkdownViewer.svelte -->
<script lang="ts">
  import { onMount } from "svelte";
  import { auth } from "$lib/store/auth.svelte";
  import { notifications } from "$lib/store/notifications.svelte";
  import MarkdownRender from "./MarkdownRender.svelte";
  import MonacoViewer from "./MonacoViewer.svelte";

  interface Props {
    loc: string;
    name: string;
    tabId: string;
  }

  let { loc, name, tabId }: Props = $props();

  let mode = $state<"view" | "edit">("view");
  let content = $state("");
  let loading = $state(true);

  async function fetchContent() {
    const response = await fetch(
      `/server/getTextFile?token=${encodeURIComponent(auth.token)}&loc=${encodeURIComponent(loc)}&name=${encodeURIComponent(name)}`,
    );
    if (!response.ok) {
      notifications.error(`Failed to load ${name}: ${response.status}`);
      loading = false;
      return;
    }
    content = (await response.json()).content ?? "";
    loading = false;
  }

  onMount(fetchContent);
</script>

<div class="flex flex-col h-full w-full bg-bg-base">
  <div class="flex items-center gap-3 px-3 h-9 border-b border-border-default text-xs text-fg-secondary">
    <span class="truncate">{name}</span>
    <span class="text-fg-muted">markdown</span>
    <div class="ml-auto inline-flex rounded border border-border-default overflow-hidden">
      <button
        type="button"
        onclick={() => (mode = "view")}
        class="px-3 h-7 text-xs transition-colors
               {mode === 'view' ? 'bg-bg-selected text-fg-accent' : 'text-fg-secondary hover:bg-bg-hover'}"
      >
        View
      </button>
      <button
        type="button"
        onclick={() => (mode = "edit")}
        class="px-3 h-7 text-xs transition-colors
               {mode === 'edit' ? 'bg-bg-selected text-fg-accent' : 'text-fg-secondary hover:bg-bg-hover'}"
      >
        Edit
      </button>
    </div>
  </div>
  <div class="flex-1 relative overflow-auto">
    {#if loading}
      <div class="absolute inset-0 flex items-center justify-center text-fg-muted text-xs">
        Loading…
      </div>
    {:else if mode === "view"}
      <MarkdownRender {content} />
    {:else}
      <MonacoViewer {loc} {name} {tabId} />
    {/if}
  </div>
</div>
```

```ts
const MARKDOWN_EXTENSIONS = new Set(["md", "markdown", "mdown", "mkd"]);

const TEXT_EXTENSIONS = new Set([
  // ... existing list MINUS "md"
  "txt", "json", "yaml", "yml", // ...
]);

export function pickViewer(extension: string): TabKind {
  const ext = extension.toLowerCase().replace(/^\./, "");
  if (MARKDOWN_EXTENSIONS.has(ext)) return "markdown";
  if (TEXT_EXTENSIONS.has(ext)) return "text";
  if (IMAGE_EXTENSIONS.has(ext)) return "image";
  if (VIDEO_EXTENSIONS.has(ext)) return "video";
  if (AUDIO_EXTENSIONS.has(ext)) return "audio";
  if (PDF_EXTENSIONS.has(ext)) return "pdf";
  return "text";
}

export function viewerIconName(kind: TabKind): string {
  switch (kind) {
    case "text":
    case "markdown":
      return "file-text";
    case "image":
      return "image";
    case "video":
      return "film";
    case "audio":
      return "music";
    case "pdf":
      return "file-type";
    default:
      return "file";
  }
}
```
